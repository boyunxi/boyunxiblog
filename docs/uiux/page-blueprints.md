# 博云隙 页面蓝图

## 0. 高级感页面规则

- 每个页面只有一个主标题、一个主视觉焦点和一个主要行动。
- 页面背景保持安静，视觉重点放在内容、排版和真实图片上。
- 首屏不使用全屏装饰性渐变、漂浮光团或过量粒子；需要氛围时仅使用低对比度材质层。
- 页面层级通过网格、字号、留白和边界表达，默认不把每个区块包成卡片。
- 高级感不等于低信息密度：信息需要完整，但按优先级分层呈现。

## 1. 前台信息架构

主导航固定为：首页、分类、标签、关于、搜索。文章页通过标签、分类、上一篇、下一篇和“继续阅读”形成闭环。URL 和现有 API 保持不变。

## 2. 首页 `/`

### 桌面结构

1. `SiteHeader`
2. 品牌首屏：站点名、清晰定位、最新文章入口；首屏高度控制在 520-620px。标题使用大字号宋体，副标题控制在一行，保留足够留白。
3. 分类入口：最多展示 5 个常用分类和“全部分类”。
4. 最新文章：以编辑部目录为主，首篇可带真实封面或局部细节图，其余使用无卡片边框的高质量列表。
5. 精选/继续阅读区：仅在存在明确内容时出现，使用单一视觉锚点，不使用空装饰区。
6. `Footer`

### 移动结构

品牌首屏控制在 420-520px，首屏底部必须露出最新文章标题或入口。分类入口横向滚动，文章列表单列，卡片摘要最多两行。

### 状态

- 无文章：说明“暂无文章”，并提供“查看关于”或后台入口（仅管理员可见）。
- 加载异常：保留品牌头部，文章区域展示错误和重试。
- 超长标题：最多三行后截断，完整标题在文章页展示。

## 3. 分类、标签和搜索页

页面统一结构：`PageHeader → 结果统计 → FilterBar → 内容列表 → Pagination/EmptyState`。筛选区与内容区之间保留 32-48px 留白，不用厚重容器包裹。

- 分类页显示分类名称、摘要统计和当前分类标记。
- 标签页显示标签名称和关联文章数。
- 搜索页支持输入中、加载中、成功、有结果、无结果、失败和清除查询。
- 搜索失败时不覆盖上一次成功结果，提供重试。
- 移动端筛选为底部抽屉，关闭后保留已选条件。

## 4. 文章页 `/posts/[slug]`

### 结构

1. 返回入口和面包屑，使用细线和小字号，不抢标题焦点。
2. 标题、分类、日期、阅读时长、浏览量。
3. 封面图（稳定比例，加载/失败占位）；图片必须具备真实主题和足够明度，不使用纯氛围背景替代内容。
4. 可选目录和正文，正文宽度 680px。
5. 标签、分享、上一篇/下一篇、继续阅读；底部采用清晰的编辑部导航，不再使用重复装饰卡片。

### 排版规则

- 长文：正文 16px、行高 1.9、段落间距 1.5em。
- 短文：允许更紧凑的段落和更少的装饰分隔。
- 代码密集文章：代码块横向滚动，复制按钮固定在代码块右上角，复制后显示成功状态。
- 标题锚点必须使用语义标题生成，不能通过手写文本模拟。

### 安全与错误

HTML/MDX 必须经过现有安全边界处理后再渲染；文章不存在或未发布时使用统一 404；图片加载失败时显示标题和替代说明。

## 5. 关于页 `/about`

使用三段式信息结构：个人简介、内容方向、联系入口。每段最多一个主要行动，品牌叙事作为辅助内容，不以连续装饰卡片堆叠页面。

## 6. 后台信息架构

主导航：概览、文章管理、分类管理、标签管理、数据看板、系统日志、站点设置。所有后台页面共享 `AdminShell + AdminTopbar + PageHeader`。

### 后台首页 `/admin`

- 顶部：欢迎语、主操作“新建文章”、次操作“管理分类”。
- 第一行：文章总数、草稿数、总浏览量、最近发布趋势；统计卡片使用细边框和轻层级，禁止大面积渐变。
- 第二行：待处理事项和最近文章表格。
- 空数据时给出创建第一篇文章的行动按钮。

### 文章管理

桌面端为表格：标题、状态、分类、更新时间、浏览量、行操作。支持搜索、状态、分类、日期筛选和分页；移动端转换为卡片并保留编辑/查看/删除操作。

### 新建/编辑文章

- 主区：标题、摘要、封面、正文编辑器。
- 侧区：分类、标签、发布状态、字数、阅读时间、最后保存时间。
- 编辑模式：编辑、预览、分屏三态；当前 Markdown/MDX 能力保持不变。
- 离开脏文档时弹出确认；保存草稿和发布使用不同主次按钮。
- 保存失败保留表单内容并定位错误；发布成功显示 Toast 并提供查看文章入口。

### 分类、标签、设置、日志

- 分类/标签：`FilterBar + DataTable + Modal/Form`，删除前显示关联内容风险。
- 设置：按站点身份、首页文案、主题、内容展示、联系信息分组。
- 日志：支持关键词、类别、时间筛选；长 meta 内容可展开查看。
- 所有 CRUD 页分别设计加载、空列表、无结果、失败和成功反馈。

### 登录页

保持品牌标题和清晰登录表单，移除大面积背景装饰。使用大留白、细边框和单一金色焦点线体现高级感。输入错误显示在字段下方，锁定状态、登录失败和提交中状态均可感知。

## 7. 页面验收清单

- 375/768/1024/1440px 无横向溢出。
- 超长中文标题、无摘要、长标签、空数据和错误数据均有稳定布局。
- 主流程可只用键盘完成。
- 所有弹窗、抽屉和移动菜单正确管理焦点。
- 关闭减少动效后仍能完成所有任务。
