# 博云隙 UI/UX 重构验收清单

## 设计系统

- [ ] 前台夜间/白昼和后台工作台均使用语义令牌。
- [ ] 颜色、字体、间距、圆角、阴影和动效没有页面级私有值。
- [ ] 每个页面只有一个主视觉焦点和一个主要行动。
- [ ] 金色视觉面积不超过页面约 5%，同一组件最多使用两种金色表达。
- [ ] 默认页面没有大面积渐变、无限循环装饰动画或卡片套卡片。
- [ ] 高级感由网格、排版、留白和材质层次表达，而非发光和大阴影。
- [ ] 功能文案使用中性、直接、可执行的表达，不依赖隐喻或情绪化词汇。
- [ ] 正文、辅助文本、状态标签通过 WCAG AA 对比度检查。
- [ ] 动效在 `prefers-reduced-motion: reduce` 下关闭或缩短。
- [ ] 按钮、输入框、图标按钮和移动端菜单项达到 44px 触控尺寸。

## 前台主流程

- [ ] 首页首屏在桌面和移动端露出有效内容入口。
- [ ] 首页、搜索、空状态和文章操作使用“查看、搜索、保存、发布”等中性动词。
- [ ] 首页能进入分类、标签、搜索和文章详情。
- [ ] 分类/标签/搜索页具备加载、失败、空结果和清除筛选状态。
- [ ] 文章页支持舒适阅读宽度、代码块横向滚动、复制反馈和上下篇导航。
- [ ] 文章封面、Markdown 图片和链接具备正确替代文本与错误处理。
- [ ] 移动导航可打开、关闭、Escape 退出并恢复焦点。

## 后台主流程

- [ ] 登录失败、锁定和提交中状态可感知。
- [ ] 概览页提供新建文章主入口和最近文章操作。
- [ ] 文章列表支持搜索、状态/分类筛选、分页和移动端卡片布局。
- [ ] 新建文章可编辑、预览、保存草稿、发布并处理离开脏文档。
- [ ] 编辑器所有图标按钮有 tooltip、`aria-label` 和焦点状态。
- [ ] 分类、标签、设置和日志页面共享表格、表单、弹窗和反馈模式。
- [ ] 删除、下线等高风险操作默认聚焦取消按钮并要求确认。

## 响应式与内容压力

- [ ] 375px、768px、1024px、1440px 均无横向滚动（代码块除外）。
- [ ] 超长标题、超长标签和无摘要不会造成布局跳动或文字溢出。
- [ ] 表格在小屏转换为可扫描卡片，操作仍可访问。
- [ ] 加载状态不会改变主要控件尺寸。

## 无障碍与回归

- [ ] 使用键盘可以完成导航、搜索、弹窗、筛选、编辑和发布。
- [ ] 焦点环清晰可见，不能只依赖颜色变化。
- [ ] 动态反馈使用正确的 `aria-live` 或 `role="alert"`。
- [ ] 外部链接、图标按钮、当前导航项和分页当前页均有语义标记。
- [ ] 视觉回归覆盖首页、文章页、搜索页、后台首页、文章列表和编辑器六条主路径。
- [ ] 现有 URL、API、认证和 Prisma 数据模型保持兼容。
